SPR Complete — Flujo de operacion

Cada fase resuelve un problema real.

El sistema opera en 6 fases secuenciales. Cada una ataca un cuello de botella especifico que hoy le cuesta tiempo, dinero y riesgo a la operacion minera.

GENERA — ejecuta en cada solicitud
APRENDE — mejora continua en background
Problema que resuelve
Como lo resuelve SPR
Problema
La informacion llega fragmentada, en formatos distintos, sin estructura.
Un supervisor envia un audio de 2 minutos, una foto borrosa y un PDF de 80 paginas. Nadie lo consolida. Se pierde contexto critico.
1
Entrada Multimodal
WhatsApp como punto unico de entrada
Audio → Texto
Transcripcion con jerga minera: "tranque", "polines", "CAEX", "ECF". MiniMax M2.5.
Imagen → Datos
Identifica equipo, EPP, zona, riesgos visibles, señaletica. Vision multimodal.
PDF / Link → Estructura
Extrae texto, tablas, especificaciones de manuales y fichas tecnicas.
Resultado unificado
Un objeto JSON con todo el contexto consolidado: tarea, equipo, zona, confianza.
APRENDE: Registra que inputs son mas comunes por zona y area
Problema
La solicitud siempre llega incompleta. Faltan datos criticos para generar un documento conforme.
El supervisor dice "necesito procedimiento para el compresor". No dice cual compresor, en que zona, para que tarea, con cuantas personas. Eso retrasa semanas.
2
Enriquecimiento de Contexto
Tres rutas para completar la informacion al 100%
Ruta A — Preguntar
Datos que solo el usuario conoce: zona, personal, turno, equipo exacto. Se pregunta via WhatsApp.
Ruta B — LLM infiere
EPP estandar, normativa aplicable, riesgos tipicos. El modelo los conoce y los completa automaticamente.
Ruta C — NotebookLM
Investigacion profunda con fuentes cruzadas. Genera resumen, preguntas clave, riesgos. En paralelo a la Ruta A.
APRENDE: Si la misma tarea siempre requiere las mismas preguntas, las hace proactivamente
Problema
Escribir un procedimiento conforme toma 3-4 semanas. La calidad depende de quien lo escribe.
Cada autor tiene su estilo. Unos citan DS 132, otros no. Unos incluyen bloqueo de energias, otros lo olvidan. No hay estandar. No hay consistencia.
3
RAG + Generacion de Documentos
Busqueda semantica, generacion IA, validacion automatica, renderizado DOCX
Paso 1: Busqueda RAG
ChromaDB busca DS 132, ECFs, RF01-RF29, manuales, procedimientos previos aprobados. Top 10 resultados.
Paso 2: Generacion LLM
MiniMax M2.5 Fine-Tuned genera el documento completo. Fallback automatico a Claude Opus 4.6.
Paso 3: Validacion auto
Claude Opus verifica: citas DS 132, RF incluidos, EPP completo, bloqueo presente, estructura SIGO.
Paso 4: Render DOCX
docxtemplater genera Procedimiento + ART + Checklist. Formato normado, listo para firmas.
APRENDE: Cada correccion de un validador se convierte en dato de entrenamiento
Problema
Los trabajadores no leen los procedimientos. 80 paginas que nadie abre.
El documento existe pero no se transfiere el conocimiento. Los operadores llegan al terreno sin saber los pasos ni los riesgos criticos. La capacitacion es inconsistente o no existe.
4
Generacion Audiovisual
NotebookLM MCP genera todo el material de capacitacion
Podcast (8 min)
Dos voces IA explican el procedimiento en formato conversacional. Para escuchar en la camioneta o antes del turno.
Quiz + Flashcards
10 preguntas de opcion multiple para certificacion. 20 tarjetas de estudio para repaso rapido.
Resumen + Mind Map
Briefing de 1 pagina para supervisores. Mapa visual de pasos, riesgos y controles interconectados.
APRENDE: Temas donde los quiz scores son bajos se refuerzan en proximos podcasts
Problema
La cadena de aprobacion es opaca. No hay trazabilidad de quien aprobo que, ni cuando.
Documentos se "aprueban" por email sin revision real. No se clasifica el riesgo. No hay escalamiento formal. SERNAGEOMIN pregunta y no hay evidencia.
5
Entrega Formal + Validacion
Paquete completo por WhatsApp + cadena de aprobacion segun riesgo
Paquete completo
DOCX + ART + Checklist + Podcast + Quiz + Fuentes citadas + Clasificacion MR. Todo en un mensaje.
MR Bajo (<8)
Supervisor de turno aprueba directamente.
MR Medio (8-31)
Supervisor + Ingeniero de Prevencion de Riesgos.
MR Alto (32-64)
Supervisor + Ing. Prev. + Gerencia General Division.
APRENDE: Cada aprobacion/rechazo alimenta metricas de calidad del modelo
Problema
No hay registro de quien esta capacitado en que. Cuando SERNAGEOMIN fiscaliza, no hay evidencia.
Sin registros de certificacion, sin versionado de documentos, sin monitoreo de cambios normativos. Multas que han aumentado +30% en 3 años.
6
Capacitacion + Auditoria
Certificacion automatica y trazabilidad total para SERNAGEOMIN
Certificacion
Trabajadores escuchan podcast, revisan docs, responden quiz. 80% o mas = certificado emitido, valido 1 año.
Auditoria continua
Monitoreo cada 6 meses: detecta cambios en DS 132, ECFs. Alerta + re-generacion automatica.
Trazabilidad total
Quien creo, valido, se capacito. Fuentes citadas. Scores de quiz. Historial inmutable versionado.
APRENDE: Cambios normativos disparan re-entrenamiento urgente del modelo
Problema
Los sistemas estaticos se degradan. La normativa cambia, el lenguaje evoluciona, los errores se repiten.
Un modelo que no aprende comete los mismos errores una y otra vez. La calidad se estanca. La confianza del usuario se pierde.

Ciclo de Aprendizaje Continuo
Fine-Tuning LoRA automatico — corre en background sin afectar produccion
Recoleccion
Docs aprobados (gold standard), correcciones de expertos, patrones de preguntas, areas de quiz score bajo.
Trigger
Cada 50 docs aprobados / Correccion senior / Cambio normativo / Cron semanal / Calidad bajo umbral.
Pipeline LoRA
Unsloth + PEFT + TRL. QLoRA 4-bit. 3-5 epochs, 2-4h GPU. Evalua ROUGE-L, compliance, estructura.
Deploy
Metricas mejoran = hot-swap en produccion. No mejoran = rollback automatico. Cero downtime.
